lego database

Lego is awesome! So when I discovered there was a data available to construct a database based on all the sets made since 1949, I could not resist having a look. So, the first step to exploring this data was to build a local database that I could query.

gihub data - for more detail

What I did

Construction of the database was done in 5 stages:

  • Install MySQL
  • Create DB and build Schema based on image above
  • Download and clean up CSV files from Rebrickable
  • Loaded CSV data into DB
  • Ran checks to ensure all was working

Why I did it

Fundamentally I did this because I like Lego! But it also privided me with a local DB to explore, test out my analytical skills and endulge my curiosity.

What I learnt

I was reminded that cleaning up the data is always required and a significant chunk of any analytical analysis. For example, I assumed (hoped) the data would be in good shape comming from a working system and in the most part it was. However, the data did not match the schema (v3) any MySQL does not like empty cells nor carrage returns which had to be addresed.
