My story

Data and programming

The foundation of my interests in data science and programming can be traced back to my studies in engineering where I used Pascal to control robots, ABAQUS for FEA and gained a deeper understanding into the maths of modelling and simulation. During my PhD, I continued to write programs to support my lab work. For example I created an image analysis software to calculate contact angles on chemically modified surfaces. This was my first insight into Object Oriented Programming using the likes of Java and C++.

While working in industry managing a team of engineers and scientists developing pathology instruments, we needed to process large quantities of time series data being generated in the lab for algorithm development. As part of this process I shifted the companies' prolific use of VBA (Excel) to MatLAB by first demonstrating proof of principle in Octave. The next development milestone was when we needed to conduct a formal verification of this pathology instrument. This required deepening up on my statistics and updating my knowledge to meet compliance to CLSI (link) standards for FDA submission, while also promoting the use of R for all statistical analysis.

Based on these foundations and experiences, I have continued to develop and apply my analytical and modelling skills in Python while developing products for the pharmaceutical and medical device industries. Further example applications of data science and analytics include:

  • Simulating the chemical degradation and mass transport in primary packaging to support design decisions to enhance drug product shelflife.
  • Integrating fluid modelling and Monte Carlo sampling to better understand and optimise the manufacture of pharmaceutical equipment.
  • Application of statistical techniques that comply with industrial standards to formally verify the performance of products.
  • Data wrangling and analysis to support investigations into colleagues' scientific hypotheses.
  • Advocate, support and mentor scientists and engineers in experimental design, statistical practices and data analysis.

Thermal modelling of boiling eggs

This investigation uses thermal modelling to simulate the Ajitsuke Tamago cooking procedure as the egg changes its stages as it solidifies.

Overview

Ajitsuke Tamago, aka Ramen eggs, are one of my favorite additions to a hearty ramen bowl. The general recipe for Ajitsuke Tamago requires adding an egg to boiling water for 7 minutes and then submerging it in ice cooled water to prevent further cooking.

For this investigation a thermal model of the egg was created using finite-differences in python and Jupyter. The model allowed custom meshes which defined the egg geometry and varying diffusion coefficients and also dynamically changed the boundary conditions based on defined criteria.

The results showed an alignment between the generally cooking method and the hardness of the composite egg parts (white and yolk) based on the temperatures reached.

Lego database

Lego is awesome! So when I discovered there was a data available to construct a database based on all the sets made since 1949, I could not resist having a look. So, the first step to exploring this data was to build a local database that I could query.

Overview

Construction of the database was done in 5 stages:

  • Install MySQL
  • Wrote SQL scripts to create DB and build Schema based on image above
  • Download and wrangling of CSV files from rebrickable
  • Loaded CSV data into DB with SQL scripts
  • Ran checks to ensure all was working

Fundamentally I did this because I like Lego! But it also provided me with a local DB to explore, test out my analytical skills and indulge my curiosity. It was a great reminded that cleaning up the data is nearly always required and a significant chunk of any analytical analysis. For example, I assumed (hoped) the data would be in good shape coming from a working system and in the most part it was. However, the data did not match the schema (v3) any MySQL does not like empty cells nor carriage returns which had to be addressed.

Lego bricks since 1949

This investigation looks into a Lego brick database to explore how lego bricks have changes since it conception in 1949.

Overview

Lego is an award winning product and has been popular for decades. Out of intrigue I create a MySQL database from rebrickable datasets. This investigation is my exploration into this database.

This investigation primarily uses R and SQL to look at the 273 different colours of bricks, trends identified in the early 2000's when the company announced it's 1.9 billion kr deficit and other observations including Fabuland Lime toilets.
